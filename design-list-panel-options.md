# Консистентные варианты панели списков дел

Дата: 2026-03-16
Область: `apps/app-mobile-web` (компонент `ListTabs`)

## 1) Что сейчас уже хорошо

- Панель списков компактная и быстро сканируется горизонтально.
- Активный список визуально выделяется нижней линией (`border-bottom`).
- Добавлено контекстное меню (ellipsis) для каждого списка.
- Есть быстрый `+` для добавления списка и бейдж количества для «Без списка».

## 2) Где ломается консистентность

1. **Смешение паттернов “tab” и “chip”**:
   - активное состояние как у tab (underline),
   - но плотность, прозрачность и расстояния — скорее как у chip.
2. **Слабая иерархия действий**:
   - основной клик по списку и меню `...` слишком визуально близки,
   - `+` не связан визуально с общей системой кнопок.
3. **Нестабильный ритм элементов**:
   - `line-height: 23px` для таба + маленькие иконки + мелкий бейдж,
   - из-за этого строка визуально «плавает» по baseline.
4. **Низкая контрастность неактивных элементов**:
   - `opacity: 0.75` для текста снижает читаемость при ярком фоне/градиентах.

## 3) Вариант A — “Чистые Tabs” (наиболее нейтральный)

Подходит, если хотите сохранить текущий характер интерфейса без радикальных изменений.

### Принципы
- Единый паттерн: все списки = табы в верхней строке.
- Активный таб: underline + semibold.
- Неактивные: без opacity, через нейтральный цвет токена.
- Меню `...` появляется только на hover/focus активного таба (или всегда только для активного).

### Мини-спека
- Высота ряда: 40px.
- Размер текста: 14px, line-height 20px.
- Горизонтальные отступы таба: 12px.
- Активный underline: 2px, full-width по кликабельной области.
- Бейдж: фиксированная высота 18px, min-width 18px, radius 999px.

### Плюсы
- Максимально знакомый UX (Notion/Jira-like).
- Легко внедрить в текущий `ListTabs` без переписывания логики.

### Риски
- Меньше “премиальности”, чем у более выразительных chip/segmented подходов.

---

## 4) Вариант B — “Segmented Chips” (более современно/premium)

Подходит, если хотите усилить визуальную идентичность и мягкость интерфейса.

### Принципы
- Вся панель — контейнер-pill с мягким фоном.
- Каждый список — chip-кнопка.
- Активный chip: фон + лёгкая внутренняя тень.
- Меню `...` внутри chip, но видно только при active/hover.

### Мини-спека
- Контейнер панели: `background: rgba(255,255,255,0.6)`, blur 8-12px, radius 14px.
- Chip: высота 32px, padding 0 12px.
- Активный chip: `background: #fff`, тень `0 1px 2px rgba(0,0,0,.08)`.
- Gap между chip: 6px.
- `+` как отдельный ghost-chip, не круглая кнопка.

### Плюсы
- Визуально консистентно с “premium” mood вашего проекта.
- Хорошо читается поверх градиентных фонов.

### Риски
- Чуть дороже по CSS и состояниям.

---

## 5) Вариант C — “Primary + Secondary Lists” (масштабируемость)

Подходит, если списков станет много, и нужна понятная навигация при росте.

### Принципы
- В первой линии: ключевые списки (например, «Мой день», «Работа», «Личное»).
- Остальные — во второй группе `Ещё` (dropdown/side sheet).
- Счётчики и `...` только для активного или pinned.

### Мини-спека
- Первая линия: до 4–5 списков максимум.
- Компонент `Ещё` показывает скрытые списки с поиском.
- Кнопка `+` рядом с `Ещё`, как action второго уровня.

### Плюсы
- Лучший вариант на перспективу (10+ списков).
- Снижает шум и горизонтальный скролл.

### Риски
- Усложнение IA и взаимодействия.

## 6) Что рекомендую для Taska прямо сейчас

**Рекомендация: начать с Варианта A + 2 акцента из Варианта B.**

Почему:
- Минимальный риск для текущей структуры `MainPage` и `ListTabs`.
- Быстрое повышение читаемости и ритма.
- Сохраняется знакомая механика табов.

### Конкретный набор улучшений (MVP 1 итерация)
1. Убрать `opacity` у `.tab`, заменить на токенизированный цвет неактивного состояния.
2. Привести высоту/line-height к единой вертикальной сетке (например, 40px row, 20px line-height).
3. Сделать `+` в том же визуальном стиле, что и табы (ghost-tab), а не круглая микрокнопка.
4. Спрятать `...` по умолчанию для неактивных табов (показывать по hover/focus).
5. Бейджу дать фиксированную геометрию и более контрастный фон.

### Следующая итерация
- Добавить контейнер панели с мягким стеклянным фоном (из Варианта B), если нужно усилить premium-ощущение.
